# 随机转盘 Chrome 扩展 - 功能设计文档（v1.3）

## 1. 项目概述

本扩展提供一个完全由用户自定义文本内容的随机转盘工具，主要用于辅助日常决策场景，例如：

- 选择午餐/晚餐菜品
- 随机挑选待办事项或兴趣活动
- 其他任意基于文本选项的随机选择


## 2. 页面结构与职责划分

采用双页面设计，将核心展示与编辑配置分离：

| 页面          | 文件名          | 主要职责                              | 打开方式                     |
|---------------|-----------------|---------------------------------------|------------------------------|
| 主界面        | popup.html      | 转盘可视化 + 结果列表展示 + 快速操作  | 点击浏览器扩展图标默认打开   |
| 设置页面      | setting.html    | 文本条目编辑 + 转动次数设置 + 方案管理 | 从 popup 跳转               |

### 2.1 popup.html（主界面） - 精简展示

**包含元素**

- **转盘可视化区域**（占据主要空间）
  - 根据当前加载的文本条目数量动态生成扇形格子
  - 每个格子显示纯文本（label）
  - 固定指针（通常位于顶部）
  - 支持旋转动画，结束后高亮选中结果

- **结果展示面板**（下方区域）
  - 按转动顺序展示本次所有结果
  - 格式示例：
    1. 麻辣烫  
    2. 日式拉面  
    3. 沙拉
  - 支持滚动（当结果较多时）

- **操作按钮区**（底部固定或转盘下方）
  - **开始转动** 按钮  
    → 读取已设置的转动次数，执行连续动画并逐条呈现结果
  - **前往设置** 按钮  
    → 打开 setting.html 进行内容与次数编辑
  - **重置** 按钮
	→ 用户点击页面其他区域使popup.html页面消失后，转盘和结果和剩余的转动次数会保持原状。在用户再次打开popup.html页面继续之前的操作
	→ 点击重置按钮后，才会清除转盘和结果。


### 2.2 setting.html（设置页面） - 编辑与管理核心

**主要功能分区**

1. **当前编辑区**
   - **文本条目列表**（动态增删）
     - 每行仅包含一个文本输入框（label）
     - 支持操作：新增一行、删除某行、上移/下移（拖拽或按钮）
     - 最小条目数：2；建议上限：50
   - **转动次数设置**
     - 数字输入框（整数）
     - 默认值：1
     - 合理范围建议：1–10（可前端限制或提示）
   - **确定** 按钮
     - 功能：将当前编辑的条目列表 + 转动次数 保存为“上一次方案”
     - 同时将当前条目同步填充至 popup 的转盘（生效）

2. **方案管理区**
   - **收藏当前** 按钮
     - 弹出输入框，要求用户输入方案名称
     - 若名称已存在，提示是否覆盖
     - 保存内容：条目列表 + 转动次数 + 方案名称 + 时间戳
   - **已收藏方案列表**
     - 列表或卡片形式展示所有收藏方案
     - 每项显示：方案名称 + 最后使用时间 + 操作
       - 使用 → 将该方案的条目与转动次数 加载到当前编辑区（可立即修改）
       - 删除 → 二次确认后移除
   - **上一次方案**（固定置顶或独立区域）
     - 显示：最后一次点击“确定”时的条目列表与转动次数
     - 操作：使用 → 加载到当前编辑区（允许修改）

**关键交互规则**

- 点击任何“使用”按钮（收藏方案或上一次方案）→ 仅填充 setting 页面的编辑区与转动次数输入框
- 填充后，用户可自由修改条目或次数
- 只有点击 **确定** 按钮，才会将当前编辑区内容同步到 popup 转盘并更新“上一次方案”
